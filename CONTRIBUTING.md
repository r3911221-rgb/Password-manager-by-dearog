Contributing to Dearog Password Manager

Thank you for your interest in contributing to Dearog Password Manager.

We welcome bug reports, feature requests, documentation improvements, code contributions, translations, testing, and security reviews from the community.

---

Code of Conduct

By participating in this project, you agree to follow the guidelines described in the CODE_OF_CONDUCT.md document.

---

Before You Start

Please ensure that you:

- Read the project documentation.
- Search existing issues before creating a new one.
- Discuss major changes before starting implementation.
- Keep pull requests focused on a single topic.

---

Development Requirements

- Android Studio (latest stable)
- JDK 21 or newer
- Android SDK
- Kotlin
- Git

---

Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a new feature branch.
4. Make your changes.
5. Test your changes.
6. Commit using clear commit messages.
7. Push your branch.
8. Open a Pull Request.

---

Coding Guidelines

- Write clean and readable Kotlin code.
- Follow the existing project architecture.
- Use meaningful class and variable names.
- Keep functions focused on a single responsibility.
- Avoid unnecessary complexity.
- Remove unused imports and dead code.
- Write comments only when they improve understanding.

---

Testing

Before submitting a Pull Request:

- Ensure the project builds successfully.
- Run unit tests.
- Run instrumentation tests where applicable.
- Verify that new features work as expected.
- Check for lint or formatting issues.

---

Documentation

If your contribution changes behavior or adds a feature, update the relevant documentation, including:

- README.md
- ARCHITECTURE.md
- CHANGELOG.md
- Any affected developer documentation

---

Pull Request Checklist

Before submitting, confirm that:

- Your code builds successfully.
- Tests pass.
- Documentation has been updated if needed.
- No sensitive information is included.
- Code follows the project's style and structure.

---

Reporting Bugs

When reporting a bug, include:

- Device model
- Android version
- App version
- Steps to reproduce
- Expected result
- Actual result
- Screenshots or logs (if available)

---

Feature Requests

Feature requests should include:

- A clear description of the problem.
- The proposed solution.
- Any alternative approaches considered.
- Additional context, if relevant.

Thank You

Every contribution—whether it's code, documentation, testing, design, or feedback—helps improve Dearog Password Manager. Thank you for being part of the project.
