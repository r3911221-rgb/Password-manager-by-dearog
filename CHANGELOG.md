Versioning

Version numbers follow this format:

MAJOR.MINOR.PATCH

Example:

- 1.0.0
- 1.1.0
- 1.2.3
- 2.0.0

MAJOR

Breaking changes or significant new functionality.

MINOR

New features that remain backward compatible.

PATCH

Bug fixes, performance improvements, and security updates.
