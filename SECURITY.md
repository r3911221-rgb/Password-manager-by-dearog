Security Policy

Thank you for helping keep Dearog Password Manager secure.

The security and privacy of our users are top priorities. We appreciate responsible disclosure of potential security issues and will work promptly to investigate and resolve valid reports.

---

Supported Versions

Only the latest stable release receives security updates.

Development and preview builds may contain experimental features and should not be considered production-ready.

Version| Security Updates
Latest Stable| ✅ Supported
Beta| ⚠ Limited
Alpha| ❌ Not Supported
Development Builds| ❌ Not Supported

---

Reporting a Vulnerability

Please report security vulnerabilities privately.

Include as much information as possible:

- Vulnerability description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Device information
- Android version
- App version
- Screenshots (if applicable)
- Logs (if applicable)

Please avoid publishing security issues publicly before they have been reviewed.

---

Response Process

After receiving a report, the Dearog team will:

1. Confirm receipt of the report.
2. Review and reproduce the issue.
3. Assess the security impact.
4. Develop and test a fix.
5. Release a security update if necessary.
6. Notify the reporter when the issue has been resolved.

---

Scope

Examples of security issues include:

- Encryption weaknesses
- Authentication bypass
- Biometric authentication issues
- PIN protection bypass
- Data leakage
- Backup security flaws
- Export vulnerabilities
- Import vulnerabilities
- Autofill vulnerabilities
- Passkey implementation issues
- Sensitive information exposure
- Clipboard security issues
- Permission-related vulnerabilities

---

Out of Scope

The following generally do not qualify as security vulnerabilities:

- UI or visual issues
- Typographical mistakes
- Feature requests
- Performance improvements
- Compatibility issues without a security impact
- Issues caused by modified operating systems or rooted devices

---

Security Principles

Dearog Password Manager is designed around the following principles:

- Privacy by default
- Strong encryption
- Offline-first architecture
- Least-privilege access
- Secure local storage
- Secure authentication
- Responsible disclosure
- Continuous security improvements

---

Disclosure Policy

Please do not publicly disclose security vulnerabilities until a fix has been released or the Dearog team has confirmed that public disclosure is appropriate.
